## Design Context

### Users
Open-source tool for anyone running OpenClaw agents. Primary user is a solo operator monitoring their agent system from a browser. Should feel immediately usable without onboarding, but scale to power users managing multiple agents across workspaces.

### Brand Personality
Warm, confident, precise. Not flashy, not sterile.

### Aesthetic Direction
- **Tone**: Editorial warm dark. DM Sans body, JetBrains Mono for data. Orange accent (hsl 24, 70%, 50%) on cool blue-gray neutrals (220deg).
- **Reference**: Linear — tight, fast, workspace switcher in small top-left area. Dense but not cluttered.
- **Anti-reference**: Generic AI dashboards (cyan-on-dark, gradient text, glassmorphism). Datadog-style visual noise. Rounded-corner card soup.
- **Theme**: Dark only. No light mode.

### Design Principles
1. **Density with clarity** — Show a lot without feeling crowded. Hierarchy through typography weight and spacing, not borders and boxes.
2. **Speed feel** — Interactions should feel instant. Tight transitions, no bounce. Linear-style snappiness.
3. **Warm, not cold** — The orange accent and tinted neutrals give it character. Lean into that warmth. Never feel clinical.
4. **Earn every pixel** — No decorative elements that don't serve function. If it doesn't help the operator, remove it.
5. **Solid surfaces** — Dropdowns, menus, overlays must have opaque backgrounds. Never transparent or glassmorphic.
